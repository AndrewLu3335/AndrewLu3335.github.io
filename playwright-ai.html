Risk conclusion

The current suite proves the read path, not the complete API surface.

Write operations are not covered, and the suite does not yet validate create, update, delete, rate-limit, concurrency, or live Strava integration behavior. These remain explicit follow-up risks rather than implied coverage.

05

Reports

HTML, JSON, screenshots, video, traces, and optional AI diagnosis support triage.

  1. Label selects templateapi, ui, cross-layer, pagination, auth, and failure-analysis route each business scenario to a stable field contract.
  2. Metadata controls executionTags such as smoke and regression never change report structure; they remain scheduling, filtering, and priority attributes.
  3. JSON template controls fieldsTeams can add a validated template in report-templates.json without modifying the renderer or executing custom JavaScript.
04

AI Failure Intelligence

AI adds investigation context, not test authority.

Playwright owns every deterministic pass/fail decision.

Evidence before inference

The analyzer receives sanitized test errors, page state, navigation, failed requests, HTTP errors, redacted console evidence, and selected screenshots. Its output is explicitly treated as a probable diagnosis that an engineer must verify.

06

Implementation Decisions

The framework keeps intelligence useful and bounded.

  1. Separate verdict from diagnosisDeterministic assertions remain the only pass/fail authority.
  2. Collect evidence only when neededFailure-specific capture reduces noise in successful runs.
  3. Sanitize before analysisURLs and common credential-like console values are removed or redacted.
  4. Require causal network evidenceAn element-not-found error alone cannot become a network classification.
